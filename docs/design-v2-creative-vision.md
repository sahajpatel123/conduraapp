# Synaptic v2 — Creative Vision: "A Living Presence"

> **Status:** Vision document. Self-generated by the design lead after deep introspection + website study.
> Created in response to user feedback: "It looks like it is purely vibe coded... does not have any fear and any alive factor."
>
> This document is the response. It is intentionally specific, opinionated, and implementable.

---

## 1. The Brand Soul (refined)

Synaptic's "alive" quality is not the Pulse. The Pulse is the visible signature of something deeper: **the desktop itself has a pulse**. The computer Synaptic runs on is no longer the user's personal machine; it is co-habited. The user can feel another presence in the room, one that watches the screen, anticipates needs, and breathes at 12 cycles per minute.

The "alive" factor for this product means: **the OS itself feels gently occupied**. Every screen has a thread that sways with the user's cursor. Every silence has pollen drifting upward. Every interactive element has a soft halo that follows the pointer. The app is not a tool the user picks up; it is an atmosphere the user inhabits.

The Pulse is a metaphor for this atmosphere. It is not the atmosphere itself. The atmosphere is the canvas of pollen, the thread that follows the cursor, the magnetic pull of buttons toward the pointer, the menu bar's quiet presence, the chat surface's editorial weight, the empty state's grain, the status bar's breathing dot. **Synaptic is felt, not used.**

## 2. The Desktop App's Unique Opportunities

| Opportunity | The "Alive" Interaction |
|---|---|
| **Persistent menu bar presence** | The status bar's Pulse brightens briefly when the user moves their cursor toward the menu bar. The agent is "looking up" — aware of the user's attention. |
| **Multi-window** | Each chat window has its own "weather" — the ambient background gradient shifts subtly with the conversation's emotional valence. A "yes" conversation is brighter; a "fix this" conversation is more focused. |
| **Long-running sessions** | The pollen drifts continuously, but the colors shift slowly over the course of a day. Morning: warm pollen. Afternoon: balanced. Evening: cooler plum. The room knows what time it is. |
| **Voice + wake word** | When the user says "hey synaptic", the Pulse in the menu bar intensifies, the chat surface's background brightens 20%, and a single synapse thread draws itself across the screen toward the cursor. The agent is "summoned". |
| **Computer use** | When the agent is about to act, a soft purple shadow trails the user's actual cursor, fading after 1.5s. The user can see the agent "considering" their cursor before acting. |
| **System-level events** | The ambient thread subtly responds to OS activity — when the user is typing elsewhere, the thread vibrates imperceptibly. The agent is *listening* even when not summoned. |
| **Idle beauty** | When the user has not interacted for 60+ seconds, the pollen drift speed halves. The agent is also resting. The room calms. The user returns to a quieter space. |

## 3. The 10 "Alive" Interactions (Implemented or Specified)

1. **Pollen Drift** — `CanvasBackground` (built). 28 particles drifting upward with sin-curve fade, plum + warm hues, respawning at the bottom edge. Continuous rAF.

2. **Cursor Halo** — `CursorHalo` (built). A soft plum ring with center dot lerps toward the OS cursor. Grows 1.6× and brightens when hovering interactive elements. Pointer-events: none — never blocks clicks.

3. **Magnetic Buttons** — `Magnetic` (built). Wraps any child. Pulls it toward the pointer within an 80px radius, strongest at the center, fading to zero at the edge. Spring-like settle. Used on the primary CTA, the brand wordmark in the topbar, and the floating ⌘K hint.

4. **Synapse Thread** — built into `CanvasBackground`. A single SVG path that sways with sine waves continuously, AND nudges toward the cursor (cx1, cx2, midY all shift based on pointer proximity). Plum color, 14% opacity. The thread is the agent's awareness line — it's always there, always moving, always slightly affected by the user.

5. **Word-by-Word Streaming** — to be added to `StreamingText`. When the agent responds, words reveal one at a time, each rising into view with a refined spring. Currently text appears all-at-once; this gives the streaming state the "alive" feel of a real conversation.

6. **Status Bar Pulse Brightening** — to be added to `StatusBar`. When the user moves their cursor toward the menu bar region, the Pulse brightens subtly (opacity 0.85 → 1.0, period slows from 5s → 7s). The agent is "looking up."

7. **Sidebar Active Trail** — to be added to `Sidebar`. When the active route changes, a faint vertical trail follows the user's last selection. A glow that fades over 1.5s.

8. **Sidebar Item Reveal on Mount** — `Sidebar` should reveal items with a 30ms stagger on first mount. Each route label slides in 4px from below. The user feels the sidebar "fill in."

9. **Ambient Gradient by Conversation** — `ChatSurface` could have a subtle background gradient that shifts based on the dominant topic of the conversation. Implementation: derive a hue from the most-recent message and apply as a 4% opacity overlay.

10. **Idle Calm** — After 60s of no user input, the pollen drift speed halves, the ambient background tints fade 20%, and the chat surface's luminance drops 1%. The room calms. Returning to the app, motion ramps back up over 1s.

## 4. The Hero Moment (the desktop equivalent of SynapseGarden)

When Synaptic is opened for the first time:

**Second 0-0.4:** The menu bar icon fades in. A plum Pulse begins breathing.

**Second 0.4-1.0:** The app window opens. The background is paper-warm cream. No UI elements yet.

**Second 1.0-2.5:** A single synapse thread draws itself across the bottom of the window — left to right, with a leading plum dot. The dot comes to rest at the center. A serif sentence fades in: "I'm here. Type when you're ready."

**Second 2.5-3.5:** The Pulse in the corner begins to breathe. Pollen particles drift upward. A subtle plum bloom appears behind the text.

**Second 3.5-6.0:** The user can type. The first character is monospace. The text becomes serif after 8 characters. As they type, the bloom behind the text intensifies slightly.

**The user feels:** The app is waiting for them. It is patient. It is alive. It is not a tool. It is an atmosphere.

## 5. The Idle Beauty (12-hour session)

**0-2 minutes (active use):** Pollen drifts at full speed. Thread sways fully. Cursor halo fully visible. The room is "alert."

**2-10 minutes (working):** No change. The user is using the app; it stays alive.

**10-30 minutes (focused elsewhere):** Pollen slows by 20%. Thread amplitude reduces by 10%. Halo opacity reduces. The room is "watching but not demanding."

**30-60 minutes (idle):** Pollen stops drifting; they hover in place, fading slowly. Thread sways with 50% amplitude. Halo only visible on direct cursor movement.

**60+ minutes (long idle):** Pollen particles slowly fade out one by one. The thread becomes still. The ambient background is at its calmest. The user returns to a space that has been "resting" with them.

**Returning from idle:** Pollen particles fade back in over 1.5s. Thread resumes sway. Halo returns. The room "wakes up" with the user.

## 6. The Confidence Signals

A premium product doesn't just work — it TELLS you it's working.

1. **The Pulse is breathing.** Always. Even when nothing else is happening. The user knows the agent is alive.

2. **The status bar dot pulses on every completed action.** A single quick brightness pulse (200ms) to acknowledge the action. The user feels the receipt.

3. **The chat surface's left-border accent grows on streaming.** The border is 1px when idle, 2px when streaming, with a subtle "breathing" animation. The user knows the agent is working without watching the text.

4. **The menu bar icon's pulse color shifts with agent state.** Idle: plum. Thinking: brighter plum. Awaiting: pulsing plum. Error: muted red, brief. The user can tell what the agent is doing without opening the app.

5. **The on-screen cursor gets a faint purple shadow when the agent is about to act.** A 200ms preview. The user sees the agent "considering" their cursor. They can cancel with a quick movement.

6. **The command surface's contextual strip updates in real time.** When the user selects a file in Finder, the contextual strip shows the file as a chip. They can invoke a command on it without copy-pasting.

7. **The Sidebar's active route has a soft glow that fades over 1s after the route changes.** "You went here" lingers visually.

8. **The keyboard ⌘K hint in the bottom-right corner breathes once when pressed.** A 200ms plum bloom. The user feels the input was received.

## 7. The Anti-Vibe-Coded Manifesto

1. **REJECT: A pulse that just breathes.**
   **PURSUE:** A pulse that breathes AND brightens when the user looks at the menu bar, AND dims when they're focused elsewhere. The pulse is responsive, not decorative.

2. **REJECT: An empty state that says "Nothing here yet."**
   **PURSUE:** An empty state with subtle ambient pattern, a faint Pulse in the background, a dot-grid that fades toward the edges, a serif italic prompt. The empty state is *alive*, not waiting.

3. **REJECT: Sidebar icons as Unicode glyphs.**
   **PURSUE:** Sidebar icons as 1.25px stroke SVG illustrations, geometric, slightly rounded joins. The icons are *designed*, not typed.

4. **REJECT: A button that changes color on hover.**
   **PURSUE:** A button that *pulls toward the cursor* on hover. The button is physically attracted to the user. The user feels they're reaching for the button and the button is reaching back.

5. **REJECT: "Loading..." text with a spinner.**
   **PURSUE:** A thin progress bar in mono that fills as the model thinks, with a heartbeat dot that scales with pause duration. The user knows the model is *thinking*, not "loading."

6. **REJECT: A static chat surface.**
   **PURSUE:** A chat surface with a hairline left-border accent that pulses while streaming, an ambient gradient that shifts with conversation topic, and a serif italic voice for the agent that uses real italics, not oblique-faked.

7. **REJECT: A menu bar icon that doesn't react to user attention.**
   **PURSUE:** A menu bar icon that brightens when the cursor is near it, dims when it's not, pulses in plum for normal breathing, and shifts to muted red for errors. The icon is the agent's face — it should have a face.

8. **REJECT: "You have 3 new notifications."**
   **PURSUE:** A toast that enters with motion from the action's source — the agent doesn't just notify, it acknowledges. The user feels the agent is *present*, not broadcasting.

9. **REJECT: A sidebar that just shows routes.**
   **PURSUE:** A sidebar that reveals items with a 30ms stagger on mount, that shows a soft glow on the active route, and that animates the collapse/expand with hairline-styled transitions. The sidebar is a *considered* navigation, not a list.

10. **REJECT: "The agent is offline."**
    **PURSUE:** A status that says "Listening" with a Pulse. "Working" with a brighter Pulse. "Awaiting" with a fast Pulse. "Error" with a red flash. The agent is always in a state, never in a void.
